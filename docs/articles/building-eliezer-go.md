# Building Eliezer Go, an Android App for Location-Based Discovery

Eliezer Go started as a simple idea: what if a mobile app could make nearby places feel a little more interactive?

The current Android build is still early, but the direction is clear. Eliezer Go combines a city-based dashboard, AR-style interaction, wallet connection flows, referrals, and progress tracking into one mobile experience. The first goal is not to push a finished product. The goal is to make the app stable enough that real people can open it, understand it, and move through the main screens without confusion.

That sounds basic, but it is the part that matters most. If the first version of an app cannot handle installation, permissions, wallet connection, navigation, and feedback from real Android devices, then larger features will only make the problem harder to debug.

## What the Current Build Focuses On

The present test build is focused on the foundation:

- Android installation and app stability
- TON wallet connection
- Wallet and progress tracking
- Frens / referral flow
- AR camera opening behavior
- Device trust checks
- Feedback from real Android phones

AR hunting is paused for now. That is intentional. It gives the project room to test the app's core systems before expanding the public gameplay layer.

## What Kind of Feedback Helps

Useful feedback is simple and practical:

- phone model
- Android version
- app version
- what screen was opened
- what the user expected
- what actually happened
- screenshot or screen recording if possible

At this stage, a clear bug report is more valuable than a large audience. The project needs real-device behavior first.
